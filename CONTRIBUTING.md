
# Contributing to Awesome Single-Cell Proteomics

First off, thank you for considering contributing to Awesome Single-Cell Proteomics! It's people like you that make this list and knowledge base a great resource for the SCP community.

## How to Contribute

We welcome contributions of new resources, tools, databases, publications, or corrections to existing content. Please follow these steps to submit your contribution:

1. **Fork the repository** to your own GitHub account.
2. **Create a new branch** for your feature or addition:
   ```bash
   git checkout -b add-new-resource
   ```
3. **Add your resource** with a brief, objective description.
4. **Commit your changes** with a clear and descriptive commit message:
   ```bash
   git commit -am 'Add [Resource Name] to [Category]'
   ```
5. **Push to the branch** on your fork:
   ```bash
   git push origin add-new-resource
   ```
6. **Create a Pull Request** from your fork to the main repository.

## Guidelines for Adding to the README

When adding a new item to the `README.md` list, please ensure it follows the existing format:
- **[Resource Name](URL)** - A brief, clear description of the resource, tool, or publication. Highlight key metrics if applicable (e.g., throughput, proteins per cell).
- Ensure the resource is placed under the most appropriate category (e.g., Sample Preparation, Software & Tools, Key Publications).
- Search the README first to make sure the resource hasn't already been added.

## Obsidian Vault Guidelines

Because this repository doubles as an Obsidian knowledge base, please keep the following in mind if you are adding or modifying individual notes in the `notes/` directory:

- **Use Templates:** If you are creating a new note, please use the appropriate template from the `templates/` folder (e.g., Method Template, Tool Template, Literature Note).
- **YAML Frontmatter:** Ensure new notes contain structured metadata (title, tags, category, url) at the top of the file.
- **Wikilinks:** Use Obsidian wikilinks (`[[note name]]`) to cross-reference related methods, tools, or concepts.
- **Callouts:** Use standard Obsidian callouts (e.g., `> [!cite]` for publications) to keep the formatting consistent.

## Code of Conduct

By participating in this project, you agree to maintain a respectful and welcoming environment for everyone. 

Thank you for helping grow the Single-Cell Proteomics community!
